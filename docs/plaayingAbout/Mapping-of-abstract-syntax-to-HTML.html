Expressions

String literal expressions

Abstract syntax tree: stringLiteral[v] where v is a sequence of characters.

Visual appearance: A black box with the string v in white monospaced characters and a light-blue border. String literals are in quotes. Numeric and boolean literals are without quotes.

The width of the box should be limited to some number of pixels (say 200) with the contents wrapping around.

Example:

"short string"
"this is a long long long long string"

-123
true

Selection

Abstract syntax: dot[v]( Exp0 ), where v is a sequence of characters.

Calls

Abstract Syntax: callClosure( Exp0, Exp1, Exp2, ..., Expn ), with n greater or equal to 0.

Visual appearance: A black box with a thin light purple border. Inside are the n+1 expression aligned horizontally. Between the first and second (or after the first if there there is no second) is a purple "!".

Before and after each expression except the first is a small dropzone. Thus the layout is roughly like this

Exp0 ! DZ Exp1 DZ Exp2 DZ

or this

Exp0 ! DZ

EXAMPLES:

f
!
fred
x
g
!

World Calls

Abstract Syntax: callWorld[Id]( Exp1, Exp2, ..., Expn ), with n greater or equal to 0.

Visual appearance: Similar to the call except there is no special treatment of the first operand and the identifier appears instead of the exclamation point. The identifier is purple.

Before and after each expression except the first is a small dropzone.

EXAMPLE:

f

New object

Abstract syntax: new( ClassType, Exp1, Exp2, ..., Expn ), with n greater or equal to 0.

Visual appearance: Similar to a call, except the left border is thick and contains a white asterisk symbol. So the layout is something like this

* ClassType ( DZ Exp1 DZ Exp2 DZ )

or this

* ClassType ( DZ )

OptType

An optional type is either of two choices, noType or a Type. When it is noType, the visual representation is simply a placeholder. When the optional type is a Type the represesentation is a white colon followed by the visual representation of the Type.

Types are discussed elsewhere.
