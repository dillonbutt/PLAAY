String literal expressions

Abstract syntax tree: stringLiteral[v] where v is a sequence of characters.

Visual appearance: A black box with the string v in white monospaced characters and a light-blue border.

The width of the box should be limited to some number of pixels (say 200) with the contents wrapping around.

HTML: <div class="stringLiteral">v</div>

Example:

abc

Calls

Abstract Syntax: call( Exp0, Exp1, Exp2, ..., Expn ), with n greater or equal to 0.

Visual appearance: A black box with a thin light purple border. Inside are the n+1 expression aligned horizontally. Between the first and second (or after the first if there there is no second) is a white open parenthesis. After the final expression (and the open parenthesis) is a which closing parenthesis.

Before and after each expression except the first is a dropzone. Thus the layout is roughly like this

Exp0 ( DZ Exp1 DZ Exp2 DZ )

or this

Exp0 ( DZ )

New object

Abstract syntax: new( ClassType, Exp1, Exp2, ..., Expn ), with n greater or equal to 0.

Visual appearance: Similar to a call, except the left border is thick and contains a white asterisk symbol. So the layout is somthing like this

* ClassType ( DZ Exp1 DZ Exp2 DZ )

or this

* ClassType ( DZ )

Method literals

Abstract syntax: method(Params, OptType Seq)

Visual appearence: A black background box with a light blue border, thick on the left side. The left side has an embedded white lower case lambda icon. (Î»). The box is divided horizontally by a dashed bluse line (as thick as the top and bottom borders). Above the line are the parameter declaration list, Params, within white parentheses, followed by the optional type. Below the line is the sequence, Seq.

Parameters

Abstract syntax: params( ParDecl1, ParDecl2, ..., ParDecln), where n is greater or equal to 0.

Visual representation: The parameters is a horizontal sequence of parameter declarations, surrounded by white parentheses and separated by drop zones. E.g.,

( DZ ParDecl1 DZ ParDecl2 DZ )

Parameter declarations

Abstract syntax: parDecl( Pattern, OptType )

Visual representation: If the optional type is present, the representation is an orange-bordered, black-backgrouned box containing the pattern, a white colon, and the type. If the type is absent, then there is no colon and a place-holder where the type can be dropped.

Patterns

Several forms of patterns will exist, but for now we stick with one kind of pattern, which is a variable introduction

Abstract Syntax: varIntro[v], where v is a sequence of characters.

Visual representation: A solid orange box with the characters v in black type.

OptType

An optional type is either of two choices, noType or a Type. When it is noType, the visual representation is simply a placeholder. When the optional type is a Type the represesentation is a white colon followed by the visual representation of the Type.

Types are discussed elsewhere.

Assignment

Abstract Syntax: Assign( Pat, Exp )

Visual representation: An thin orange-bordered, black-backgrounded box containing horizontally: the pattern, a bold white := and the expression.

If-then-else

Abstract syntax: if( Exp, Seq, Seq)

Visual representation: A purple-bordered, black-backgrounded box with a white question mark icon in the left border near the top. Vertically within the box are the expression, Exp, a dashed purple line, the first sequence, a solid dashed line, and the second sequence.

While loops

Abstract Syntax: while( Exp, Seq ).

Visual representation: Much like the if-then-else except without the solid line or the second sequence. The icond is the bliss symbol for rotation, a circular arrow.

Sequences

Abstract syntax: seq( SeqMember, SeqMember, ... ) , with 0 or more sequence members

Visual repreesentation: The sequence members are stacked vertically with drop zones before and after each one.

Each sequence member is either an expression or a variable declaration.
